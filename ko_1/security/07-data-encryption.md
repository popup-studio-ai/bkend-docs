# 데이터 암호화

> bkend에서 데이터를 보호하는 암호화 방식을 안내합니다.

## 개요

bkend는 전송 중 암호화와 저장 시 암호화를 모두 적용하여 데이터를 보호합니다.

---

## 전송 중 암호화 (In Transit)

| 항목 | 설명 |
|------|------|
| **프로토콜** | TLS 1.2 이상 |
| **대상** | 모든 API 통신 |
| **적용** | HTTPS 강제 (HTTP 요청 자동 리다이렉트) |

모든 API 요청은 HTTPS를 통해 암호화됩니다.

---

## 저장 시 암호화 (At Rest)

| 항목 | 설명 |
|------|------|
| **데이터베이스** | MongoDB Atlas 기본 암호화 (AES-256) |
| **파일 저장소** | AWS S3 서버 사이드 암호화 (SSE-S3) |

---

## 민감 데이터 처리

| 데이터 | 처리 방식 |
|--------|----------|
| **비밀번호** | bcrypt 해싱 (원본 미저장) |
| **API Key** | SHA256 해싱 (원본 미저장) |
| **Refresh Token** | 세션별 저장, 갱신 시 이전 토큰 무효화 |
| **OAuth Token** | RBAC 필드 필터링으로 노출 제한 |
| **MFA Secret** | 본인만 접근 가능 |

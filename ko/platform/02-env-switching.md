# 환경 전환하기

> 콘솔에서 환경을 전환하고 관리하는 방법을 안내합니다.

## 개요

bkend 콘솔의 상단 바(TopBar)에서 환경을 전환할 수 있습니다. 각 환경은 독립된 데이터베이스와 설정을 가지므로, 작업할 환경을 정확히 선택하세요.

---

## 환경 전환하기

### 콘솔 상단 바 구조

콘솔 상단 바는 다음과 같은 계층 구조로 표시됩니다.

```
Organization / Project / Environment
```

### 환경 전환 방법

1. 콘솔 상단 바에서 **Environment** 드롭다운을 클릭하세요.
2. 드롭다운에 현재 프로젝트의 모든 환경이 표시됩니다.
3. 전환할 환경을 선택하세요.

```mermaid
flowchart LR
    A[상단 바 Environment 클릭] --> B[환경 목록 표시]
    B --> C[환경 선택]
    C --> D[해당 환경으로 전환]
    D --> E[데이터/설정 갱신]
```

### 환경 드롭다운 표시 정보

각 환경 항목에는 다음 정보가 표시됩니다.

| 요소 | 설명 |
|------|------|
| **환경 이름** | dev, staging, prod 등 |
| **상태 배지** | creating, ready, active, failed, deleting |
| **선택 표시** | 현재 선택된 환경에 체크 아이콘 |

---

## Environments 페이지

프로젝트의 **Environments** 메뉴에서 모든 환경을 한눈에 확인할 수 있습니다.

### 환경 카드 정보

각 환경은 카드 형태로 표시되며 다음 정보를 포함합니다.

| 항목 | 설명 |
|------|------|
| **환경 이름** | dev, staging, prod, custom |
| **배포 상태** | 상태 배지로 표시 |
| **클라우드** | AWS, GCP 등 클라우드 제공자 |
| **리전** | 배포 리전 (예: ap-northeast-2) |
| **클러스터 정보** | 클러스터 이름과 타입 (staging/prod만 표시) |
| **생성일** | 환경 생성 날짜 |
| **에러 메시지** | 배포 실패 시 에러 내용 표시 |

### 상태 배지 색상

| 상태 | 색상 | 의미 |
|------|------|------|
| `creating` | 노란색 | 프로비저닝 진행 중 |
| `ready` | 파란색 | 사용 준비 완료 |
| `active` | 초록색 | 활성 운영 중 |
| `failed` | 빨간색 | 프로비저닝 실패 |
| `deleting` | 회색 | 삭제 진행 중 |

> ⚠️ **주의** - `failed` 상태의 환경은 빨간색 박스로 에러 메시지가 표시됩니다. 에러 내용을 확인하고 필요시 환경을 다시 생성하세요.

---

## 환경 전환 시 동작

환경을 전환하면 다음이 변경됩니다.

| 변경 항목 | 설명 |
|----------|------|
| **테이블 목록** | 해당 환경의 테이블만 표시 |
| **User 목록** | 해당 환경의 User만 표시 |
| **파일 목록** | 해당 환경의 파일만 표시 |
| **API Key** | 해당 환경의 API Key만 표시 |
| **활동 로그** | 해당 환경의 로그만 표시 |

> 💡 **Tip** - 프로젝트를 전환하면 환경도 자동으로 초기화됩니다. 첫 번째 환경이 자동 선택됩니다.

# 조직 관리

{% hint style="info" %}
💡 Organization은 bkend의 최상위 리소스로, 프로젝트와 팀원을 하나의 단위로 관리합니다.
{% endhint %}

## 개요

Organization은 프로젝트, 팀, 결제를 묶는 최상위 단위입니다. 하나의 Organization 안에 여러 프로젝트를 생성하고, 팀원을 초대하여 함께 관리할 수 있습니다.

***

## 조직 생성하기

1. 콘솔에 로그인하세요.
2. **조직 생성** 버튼을 클릭하세요.
3. 다음 정보를 입력하세요.

| 필드 | 필수 | 설명 |
|------|:----:|------|
| **조직 이름** | ✅ | 1~100자. 조직의 표시 이름 |
| **표시 이름** | - | 최대 200자. 대시보드에 표시되는 이름 |
| **설명** | - | 최대 500자. 조직 설명 |

4. **생성**을 클릭하세요.

## 조직 홈 대시보드

조직을 선택하면 홈 대시보드가 표시됩니다.

<!-- 📸 IMG: 조직 홈 대시보드 -->
![조직 목록](../.gitbook/assets/2026-02/console-org-list.png)

### 빠른 통계

| 항목 | 설명 |
|------|------|
| 프로젝트 수 | 조직 내 프로젝트 총 수 |
| 팀 멤버 수 | 조직에 참여 중인 멤버 수 |
| 스토리지 사용량 | 전체 파일 스토리지 사용량 |
| 테이블 수 | 전체 프로젝트의 테이블 총 수 |

### 빠른 액션

- **새 프로젝트 생성** — 프로젝트를 추가합니다
- **팀 멤버 초대** — 팀원을 초대합니다
- **프로젝트 보기** — 프로젝트 목록으로 이동합니다
- **조직 설정** — 조직 설정 페이지로 이동합니다

***

## 조직 설정 변경하기

1. 사이드바에서 **설정**을 클릭하세요.
2. **일반** 탭에서 조직 이름, 표시 이름, 설명을 수정하세요.
3. **저장**을 클릭하세요.

### 조직 정보 확인

설정 페이지에서 다음 정보를 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| Organization ID | 고유 식별자 (복사 가능) |
| Slug | URL에 사용되는 식별자 |
| 생성 날짜 | 조직이 생성된 날짜 |
| 요금제 | 현재 적용 중인 플랜 |

***

## 감사 로그 확인하기

1. **설정** > **감사 로그** 탭을 클릭하세요.
2. 필터를 사용하여 특정 활동을 검색하세요.

| 필터 | 옵션 |
|------|------|
| **검색** | 메시지 텍스트로 검색 |
| **엔티티 타입** | organization, project, environment, table, access_token, invitation, member |
| **액션** | created, updated, deleted, deployed, accepted, revoked |

***

## 조직 소유권 이전하기

{% hint style="danger" %}
🚨 **위험** — 소유권을 이전하면 현재 Owner는 Admin으로 변경됩니다.
{% endhint %}

1. **설정** > **위험 영역** 탭을 클릭하세요.
2. **소유권 이전** 섹션에서 새 소유자를 선택하세요.
3. 확인 텍스트 `transfer`를 입력하세요.
4. **이전**을 클릭하세요.

***

## 조직 삭제하기

{% hint style="danger" %}
🚨 **위험** — 조직을 삭제하면 모든 프로젝트, 환경, 데이터가 영구적으로 삭제됩니다. 이 작업은 되돌릴 수 없습니다.
{% endhint %}

1. **설정** > **위험 영역** 탭을 클릭하세요.
2. **조직 삭제** 섹션에서 조직 이름을 입력하세요.
3. **삭제**를 클릭하세요.

***

## 조직 탈퇴하기

Owner가 아닌 멤버는 조직에서 탈퇴할 수 있습니다.

1. **설정** > **위험 영역** 탭을 클릭하세요.
2. 확인 텍스트 `leave`를 입력하세요.
3. **탈퇴**를 클릭하세요.

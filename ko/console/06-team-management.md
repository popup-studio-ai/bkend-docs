# 팀원 관리

{% hint style="info" %}
💡 Organization에 팀원을 초대하고 역할을 부여하여 함께 프로젝트를 관리하세요.
{% endhint %}

## 개요

팀 관리는 Organization 레벨에서 수행됩니다. 팀원을 이메일로 초대하고, 역할을 부여하여 접근 권한을 제어합니다.

***

## 역할

| 역할 | 설명 |
|------|------|
| **Owner** | 모든 권한. 조직 삭제, 소유권 이전 가능 |
| **Admin** | 대부분의 관리 권한. 소유권 이전 제외 |
| **Member** | 읽기 및 기본 기능 |
| **Billing** | 결제 관련 기능 |

***

## 팀원 초대하기

1. 사이드바에서 **팀**을 클릭하세요.
2. **멤버 초대** 버튼을 클릭하세요.
3. 다음 정보를 입력하세요.

| 필드 | 필수 | 설명 |
|------|:----:|------|
| **이름** | ✅ | 초대할 멤버의 이름 |
| **이메일** | ✅ | 초대 링크가 발송될 이메일 |
| **역할** | ✅ | Owner, Admin, Member, Billing (기본값: Member) |

4. **초대**를 클릭하세요.

<!-- 📸 IMG: 팀원 초대 모달 -->
![팀원 초대](../.gitbook/assets/2026-02/console-team-invite.png)

{% hint style="info" %}
💡 초대 이메일은 만료 기한이 있습니다. 만료 전에 수락하지 않으면 재발송이 필요합니다.
{% endhint %}

***

## 팀원 목록 조회하기

<!-- 📸 IMG: 팀원 목록 화면 -->
![팀원 목록](../.gitbook/assets/2026-02/console-team-list.png)

### 검색 및 필터

| 필터 | 옵션 |
|------|------|
| **검색** | 이름 또는 이메일로 검색 |
| **역할** | Owner, Admin, Member, Billing, All |
| **상태** | Active, Pending, All |

### 활성 멤버 테이블

| 컬럼 | 설명 |
|------|------|
| 멤버 | 아바타, 이름, 이메일 |
| 역할 | Owner / Admin / Member / Billing |
| 상태 | Active |
| 참여일 | 조직에 참여한 날짜 |
| 액션 | 역할 변경, 제거 (Admin/Owner만) |

### 대기 중인 초대 테이블

| 컬럼 | 설명 |
|------|------|
| 이메일 | 초대된 이메일 주소 |
| 역할 | 부여된 역할 |
| 상태 | Pending / Accepted / Cancelled / Expired |
| 만료일 | 초대 만료 날짜 |
| 액션 | 재발송 / 취소 (Admin/Owner만) |

***

## 역할 변경하기

1. 팀원 목록에서 해당 멤버의 **더보기(⋯)** 메뉴를 클릭하세요.
2. **역할 변경**을 선택하세요.
3. 새 역할을 선택하세요.
4. **변경**을 클릭하세요.

{% hint style="warning" %}
⚠️ Owner만 다른 멤버의 역할을 변경할 수 있습니다. Admin은 Member와 Billing 역할만 변경할 수 있습니다.
{% endhint %}

***

## 팀원 제거하기

{% hint style="danger" %}
🚨 **위험** — 팀원을 제거하면 해당 멤버는 조직의 모든 프로젝트에 접근할 수 없습니다.
{% endhint %}

1. 팀원 목록에서 해당 멤버의 **더보기(⋯)** 메뉴를 클릭하세요.
2. **멤버 제거**를 선택하세요.
3. 멤버의 이메일을 입력하여 확인하세요.
4. **제거**를 클릭하세요.

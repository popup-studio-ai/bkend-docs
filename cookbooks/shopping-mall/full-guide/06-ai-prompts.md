# 06. AI 프롬프트 모음

{% hint style="info" %}
💡 쇼핑몰 프로젝트의 각 기능을 AI에게 자연어로 요청하는 프롬프트 레퍼런스입니다. MCP가 연결된 AI 클라이언트에서 아래 프롬프트를 그대로 사용하세요.
{% endhint %}

## 이 장에서 배우는 것

- 테이블 생성 프롬프트 (구조 확인 포함)
- 상품 등록/수정/삭제/재고 관리 프롬프트
- 장바구니 관리 프롬프트
- 주문 생성 및 상태 변경 프롬프트
- 리뷰 작성 및 평점 관리 프롬프트
- 여러 기능을 조합하는 복합 시나리오

***

## 기본 설계 참조

AI가 매번 다른 구조를 만들 수 있습니다. 아래 표를 참고하여 AI가 제안하는 구조가 적절한지 확인하세요.

### products (상품)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| name | 상품 이름 | "무선 이어폰" |
| description | 상품 설명 | "블루투스 5.0..." |
| price | 가격 | 29900 |
| category | 카테고리 | "전자제품" |
| stock | 재고 수량 | 50 |
| imageUrl | 상품 사진 URL | (업로드 후 연결) |
| isActive | 판매 중 여부 | true / false |

### carts (장바구니)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| productId | 어떤 상품인지 | (상품 ID) |
| quantity | 수량 | 2 |

### orders (주문)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| items | 주문 상품 목록 | [{상품명, 가격, 수량}] |
| totalPrice | 총 금액 | 59800 |
| status | 주문 상태 | "pending" / "confirmed" / "shipped" / "delivered" |
| shippingAddress | 배송 주소 | "서울시 강남구..." |
| recipientName | 수령인 이름 | "김고객" |
| recipientPhone | 수령인 연락처 | "010-1234-5678" |

### reviews (리뷰)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| productId | 리뷰 대상 상품 | (상품 ID) |
| orderId | 구매한 주문 | (주문 ID) |
| rating | 별점 (1~5) | 5 |
| content | 리뷰 내용 | "음질이 좋아요" |

### 주문 상태 표현 매핑

| 말하는 표현 | 저장되는 값 |
|------------|-----------|
| 주문 대기 | pending |
| 주문 확인 | confirmed |
| 배송 중 | shipped |
| 배달 완료 | delivered |

## 테이블 만들기

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"쇼핑몰에서 상품을 관리하고 싶어요. 상품 이름, 설명, 가격, 카테고리, 재고 수량, 상품 사진, 판매 여부를 저장할 수 있게 해주세요. 만들기 전에 어떤 구조로 만들지 먼저 보여주세요."
{% endhint %}

{% hint style="info" %}
💡 AI가 구조를 보여주면, 위 [기본 설계 참조](#기본-설계-참조)와 비교해서 확인하세요. 괜찮으면 "네, 만들어주세요"라고 하면 됩니다.
{% endhint %}

{% hint style="success" %}
✅ **나머지 테이블도 같은 방식으로 요청하세요**

- "장바구니 기능을 만들고 싶어요. 어떤 상품인지, 몇 개 담았는지를 저장할 수 있게 해주세요. 만들기 전에 구조를 먼저 보여주세요."
- "주문 관리 기능을 만들고 싶어요. 주문 상품 목록, 총 금액, 주문 상태, 배송 주소, 수령인 정보를 저장할 수 있게 해주세요. 만들기 전에 구조를 먼저 보여주세요."
- "상품 리뷰 기능을 만들고 싶어요. 어떤 상품의 리뷰인지, 별점(1~5), 리뷰 내용을 저장할 수 있게 해주세요. 만들기 전에 구조를 먼저 보여주세요."
{% endhint %}

### 구조 확인

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"상품 데이터가 어떤 구조로 저장되어 있는지 보여주세요."
{% endhint %}

{% hint style="success" %}
✅ **전체 확인**

"지금 만들어진 데이터 저장소가 어떤 것들이 있는지 보여주세요."
{% endhint %}

## 상품 관리

### 상품 등록

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"새 상품을 등록해주세요. 프리미엄 면 티셔츠, 부드러운 100% 면 소재 기본 티셔츠예요. 가격 29,000원, 카테고리는 의류, 재고 100개입니다."
{% endhint %}

{% hint style="success" %}
✅ **다양한 카테고리 상품 등록**

- "전자기기 카테고리에 블루투스 이어폰을 등록해주세요. 가격 45,000원, 재고 200개요."
- "식품 카테고리에 유기농 꿀 1kg을 등록해주세요. 가격 32,000원, 재고 50개요."
{% endhint %}

### 상품 수정

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프리미엄 면 티셔츠 가격을 25,000원으로 내려주세요."
{% endhint %}

{% hint style="success" %}
✅ **판매 중지/재개**

- "프리미엄 면 티셔츠를 판매 중지해주세요."
- "프리미엄 면 티셔츠를 다시 판매 시작해주세요."
{% endhint %}

### 상품 삭제

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프리미엄 면 티셔츠 상품을 삭제해주세요."
{% endhint %}

{% hint style="warning" %}
⚠️ 삭제된 상품은 복구할 수 없습니다. 일시적으로 판매를 중단하려면 "판매 중지해주세요"라고 요청하세요.
{% endhint %}

### 상품 사진 업로드

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프리미엄 면 티셔츠에 상품 사진을 추가하고 싶어요. 이미지 파일을 업로드해주세요."
{% endhint %}

AI가 이미지를 업로드하고, 상품 사진을 자동으로 연결합니다.

### 재고 관리

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

- "재고가 10개 이하인 상품 목록을 보여주세요."
- "프리미엄 면 티셔츠 재고를 150개로 늘려주세요."
{% endhint %}

### 카테고리별 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

- "의류 카테고리 상품만 보여주세요."
- "전자기기 중에서 5만원 이하 상품을 가격이 낮은 순서로 보여주세요."
- "판매 중인 상품 전체를 보여주세요."
{% endhint %}

{% endhint %}

### 수량 변경

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"장바구니에 있는 프리미엄 면 티셔츠 수량을 3개로 바꿔주세요."

### 장바구니 항목 삭제

{% hint style="success" %}

AI가 장바구니에 담긴 모든 상품을 제거합니다.

### 장바구니 확인

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"내 장바구니에 뭐가 있는지 보여주세요."
{% endhint %}

## 주문 관리

### 주문하기 (장바구니 -> 주문)

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"장바구니에 담긴 상품으로 주문해주세요. 배송지는 서울시 서초구 반포대로 45, 수령인 김고객, 연락처 010-1234-5678이에요. 주문 후 장바구니는 비워주세요."
{% endhint %}

AI가 다음을 자동으로 처리합니다.

1. 장바구니 확인
2. 상품 가격 확인 및 총 금액 계산
3. 주문 생성
4. 장바구니 비우기

### 주문 상태 변경

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

- "가장 최근 주문 상태를 '주문 확인'으로 바꿔주세요."
- "가장 최근 주문 상태를 '배송 중'으로 바꿔주세요."
- "가장 최근 주문 상태를 '배달 완료'로 바꿔주세요."
{% endhint %}

{% hint style="warning" %}
⚠️ 주문 상태는 순서대로만 변경할 수 있습니다. '주문 대기'에서 바로 '배달 완료'로 건너뛸 수 없습니다.
{% endhint %}

### 주문 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

- "내 주문 내역을 최근 순서로 보여주세요."
- "배송 중인 주문만 보여주세요."
- "가장 최근 주문의 상세 정보를 보여주세요."
{% endhint %}

## 리뷰

### 리뷰 작성

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프리미엄 면 티셔츠에 별점 5점으로 리뷰를 남겨주세요. 내용은 '품질이 정말 좋아요! 면 소재가 부드럽고 핏도 편합니다.'로 해주세요."
{% endhint %}

{% hint style="info" %}
💡 리뷰는 배달 완료된 주문의 상품에 대해서만 작성하는 것을 권장합니다.
{% endhint %}

### 별점 수정

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"내가 쓴 티셔츠 리뷰 별점을 4점으로 바꾸고, 내용도 '전체적으로 만족합니다. 다만 사이즈가 약간 크네요.'로 수정해주세요."
{% endhint %}

### 상품별 리뷰 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

- "프리미엄 면 티셔츠에 달린 리뷰를 보여주세요."
- "프리미엄 면 티셔츠의 별점 5점 리뷰만 보여주세요."
{% endhint %}

### 평균 별점 확인

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프리미엄 면 티셔츠의 리뷰를 요약해주세요. 평균 별점이 몇 점인지, 별점별로 몇 개씩인지 알려주세요."
{% endhint %}

AI가 리뷰를 조회하고, 평균 별점과 분포를 자동으로 계산하여 보여줍니다.

## 복합 시나리오

여러 기능을 조합하는 프롬프트 예시입니다. AI가 여러 작업을 순차적으로 처리합니다.

### 시나리오 1: 신규 상품 등록 + 사진 업로드

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"전자기기 카테고리에 새 상품을 등록해주세요. 상품명은 '무선 충전 패드', 가격 35,000원, 재고 50개예요. 상품 사진도 업로드해주세요."
{% endhint %}

AI가 상품을 등록한 뒤, 사진을 업로드하고 자동으로 연결합니다.

### 시나리오 2: 장바구니 -> 주문 -> 장바구니 비우기

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"장바구니에 있는 상품으로 주문해주세요. 배송지는 부산시 해운대구 해운대로 100이에요. 주문 후 장바구니는 비워주세요."
{% endhint %}

AI가 장바구니 확인, 총 금액 계산, 주문 생성, 장바구니 비우기를 순서대로 처리합니다.

### 시나리오 3: 주문 완료 후 리뷰 작성

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"최근 배달 완료된 주문에서 프리미엄 면 티셔츠에 별점 5점 리뷰를 남겨주세요. 내용은 '기대 이상이에요. 재구매 의사 있습니다.'"
{% endhint %}

AI가 배달 완료된 주문을 찾아 해당 상품에 리뷰를 작성합니다.

### 시나리오 4: 재고 부족 상품 확인 + 판매 중지

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"재고가 5개 이하인 상품을 찾아서 전부 판매 중지해주세요."
{% endhint %}

AI가 재고 부족 상품을 조회한 뒤, 각 상품의 판매를 중지합니다.

### 시나리오 5: 일일 운영 리포트

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"오늘 주문 현황을 요약해주세요. 상태별 건수와 총 매출도 알려주세요."
{% endhint %}

AI가 오늘의 주문을 조회하고, 상태별 분류 및 매출을 정리하여 보여줍니다.

### 시나리오 6: 상품 리뷰 분석

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프리미엄 면 티셔츠의 최근 리뷰를 분석해서 고객이 좋아하는 점, 아쉬운 점, 개선할 점을 정리해주세요."
{% endhint %}

AI가 리뷰를 조회하고, 긍정/부정 의견과 개선 제안을 정리하여 보여줍니다.

## 프롬프트 작성 팁

| 팁 | 설명 | 예시 |
|-----|------|------|
| 구체적으로 | 상품명, 가격, 수량 등을 명시하세요 | "티셔츠 등록해줘" -> "프리미엄 면 티셔츠를 29,000원에 등록해주세요" |
| 조건 포함 | 조건을 자연어로 말하세요 | "재고 10개 이하, 의류만" |
| 단계 연결 | 여러 작업을 한 문장으로 요청하세요 | "장바구니 상품으로 주문하고, 장바구니 비워주세요" |
| 결과 요청 | 원하는 출력 형태를 지정하세요 | "표로 정리해주세요", "요약해주세요" |
| 구조 확인 | 새로 만들 때는 먼저 확인하세요 | "만들기 전에 어떤 구조로 만들지 먼저 보여주세요" |

{% hint style="info" %}
💡 AI는 대화 흐름을 기억합니다. 이전에 등록한 상품을 "방금 등록한 상품"으로 지칭해도 됩니다.
{% endhint %}

# 프로젝트 관리

{% hint style="info" %}
💡 Project는 하나의 서비스 또는 앱에 대응하는 리소스 단위입니다.
{% endhint %}

## 개요

Project는 Organization 내에서 하나의 서비스를 구성하는 단위입니다. 프로젝트를 생성하면 `dev` 환경이 자동으로 프로비저닝되며, 데이터베이스, 인증, 스토리지를 바로 사용할 수 있습니다.

***

## 프로젝트 생성하기

1. 사이드바에서 **프로젝트**를 클릭하세요.
2. **프로젝트 생성** 버튼을 클릭하세요.
3. 다음 정보를 입력하세요.

| 필드 | 필수 | 설명 |
|------|:----:|------|
| **프로젝트 이름** | ✅ | 1~100자. 프로젝트 표시 이름 |
| **프로젝트 Slug** | ✅ | URL 식별자 (이름에서 자동 생성) |
| **리전** | ✅ | 데이터 저장 위치 (예: Seoul) |
| **클라우드** | ✅ | 클라우드 제공자 (예: AWS) |

4. **생성**을 클릭하세요.

<!-- 📸 IMG: 프로젝트 생성 화면 -->
![프로젝트 생성](../.gitbook/assets/2026-02/console-project-create.png)

{% hint style="warning" %}
⚠️ 프로젝트를 생성하면 `dev` 환경이 자동으로 프로비저닝됩니다. **Active** 상태가 될 때까지 약 30초를 기다리세요.
{% endhint %}

{% hint style="info" %}
💡 현재 Organization당 최대 **3개** 프로젝트를 생성할 수 있습니다.
{% endhint %}

***

## 프로젝트 목록 조회하기

사이드바에서 **프로젝트**를 클릭하면 프로젝트 목록이 카드 형태로 표시됩니다.

<!-- 📸 IMG: 프로젝트 목록 화면 -->
![프로젝트 목록](../.gitbook/assets/2026-02/console-project-list.png)

### 검색 및 필터

| 필터 | 설명 |
|------|------|
| **검색** | 이름, slug, 설명, ID로 검색 |
| **상태** | All Status, Active, Inactive |
| **리전** | 프로젝트의 리전별 필터 |
| **정렬** | 생성일/수정일/이름 + 오름차순/내림차순 |

***

## 프로젝트 대시보드

프로젝트를 클릭하면 대시보드가 표시됩니다.

### 빠른 통계

| 항목 | 설명 |
|------|------|
| Database | 테이블 수 |
| Storage | 스토리지 사용량 |
| Authentication | 사용자 수 |

### 프로젝트 정보

| 항목 | 설명 |
|------|------|
| Project ID | 고유 식별자 (복사 가능) |
| Region | 데이터 저장 리전 |
| Cloud | 클라우드 제공자 |
| 생성 날짜 | 프로젝트 생성일 |
| Environments | 환경 수 |

### 빠른 액션

- **Database 관리** — 데이터베이스 페이지로 이동
- **Authentication 관리** — 인증 페이지로 이동
- **File Storage 관리** — 스토리지 페이지로 이동
- **Environments 관리** — 환경 페이지로 이동

***

## 프로젝트 삭제하기

{% hint style="danger" %}
🚨 **위험** — 프로젝트를 삭제하면 모든 환경, 테이블, 데이터, 파일이 영구적으로 삭제됩니다.
{% endhint %}

1. 프로젝트 목록에서 해당 프로젝트의 **더보기(⋯)** 메뉴를 클릭하세요.
2. **Delete**를 선택하세요.
3. 프로젝트 이름을 입력하여 확인하세요.
4. **삭제**를 클릭하세요.

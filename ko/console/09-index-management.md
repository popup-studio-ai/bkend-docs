# 인덱스 관리

{% hint style="info" %}
💡 인덱스를 생성하여 데이터 조회 성능을 향상시키는 방법을 안내합니다.
{% endhint %}

## 개요

인덱스는 특정 컬럼의 조회 속도를 향상시킵니다. 자주 검색하거나 필터링하는 컬럼에 인덱스를 생성하면 데이터 조회가 빨라집니다.

***

## 인덱스 목록 확인하기

1. **데이터베이스** → 테이블 선택 → **인덱스** 탭을 클릭하세요.
2. 현재 생성된 인덱스를 확인합니다.

***

## 인덱스 생성하기

{% tabs %}
{% tab title="콘솔" %}
1. 인덱스 탭에서 **인덱스 추가** 버튼을 클릭하세요.
2. 다음 정보를 입력하세요.

| 필드 | 설명 |
|------|------|
| **인덱스 이름** | 인덱스 식별자 |
| **대상 컬럼** | 인덱스를 적용할 컬럼 (복합 가능) |
| **고유** | UNIQUE 인덱스 여부 |

3. **생성**을 클릭하세요.
{% endtab %}
{% tab title="MCP" %}
AI 도구에서 자연어로 요청하세요.

```text
"posts 테이블의 author_id 컬럼에 인덱스를 만들어줘"
```
{% endtab %}
{% endtabs %}

{% hint style="info" %}
💡 복합 인덱스는 여러 컬럼을 조합하여 생성합니다. 자주 함께 검색되는 컬럼을 하나의 인덱스로 묶으면 효과적입니다.
{% endhint %}

***

## 인덱스 삭제하기

1. 삭제할 인덱스의 **삭제** 아이콘을 클릭하세요.
2. 확인 후 인덱스가 삭제됩니다.

{% hint style="warning" %}
⚠️ 인덱스를 삭제하면 해당 컬럼의 조회 성능이 저하될 수 있습니다. 데이터는 영향을 받지 않습니다.
{% endhint %}

***

## 인덱스 권장 사항

| 상황 | 인덱스 권장 |
|------|:---------:|
| 자주 WHERE 절에 사용되는 컬럼 | ✅ |
| 정렬(ORDER BY)에 자주 사용되는 컬럼 | ✅ |
| 유니크 제약이 필요한 컬럼 | ✅ |
| 거의 모든 값이 동일한 컬럼 | ❌ |
| 자주 변경되는 컬럼 | △ (쓰기 성능 저하 가능) |

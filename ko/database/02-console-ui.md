# 콘솔에서 DB 관리하기

> bkend 콘솔의 Database 화면에서 테이블과 데이터를 시각적으로 관리하는 방법을 안내합니다.

## 개요

bkend 콘솔의 **Database** 메뉴에서 테이블 생성, 스키마 관리, 데이터 조회/편집, 인덱스 관리를 수행할 수 있습니다.

---

## Database 페이지 접근하기

1. 콘솔에서 프로젝트를 선택하세요.
2. Sidebar의 **Database** 메뉴를 클릭하세요.
3. 상단의 환경 탭에서 원하는 환경(dev, staging, prod)을 선택하세요.

---

## 테이블 목록 화면

### 화면 구성

| 영역 | 설명 |
|------|------|
| **환경 탭** | dev, staging, prod 환경 전환 (배포 상태 배지 표시) |
| **테이블 목록** | 테이블명, 상태, 행 수, 수정일 표시 |
| **검색** | 테이블명으로 검색 |
| **새로고침** | 목록 새로고침 버튼 |
| **테이블 생성** | **+ 테이블 생성** 버튼 (Admin 이상) |

### 테이블 목록 컬럼

| 컬럼 | 설명 |
|------|------|
| **이름** | 테이블명과 아이콘 |
| **상태** | 배포 상태 배지 (`deployed`, `deploying`, `updating`, `failed`) |
| **행 수** | 데이터 행 개수 (K, M 단위 표시) |
| **수정일** | 마지막 수정 시간 |
| **액션** | 보기, 삭제 메뉴 |

---

## 테이블 생성 다이얼로그

**+ 테이블 생성** 버튼을 클릭하면 생성 다이얼로그가 열립니다.

### 입력 항목

| 항목 | 설명 | 규칙 |
|------|------|------|
| **테이블 이름** | 테이블 식별자 | 소문자 영문으로 시작, `[a-z][a-z0-9_]*` |
| **필드** | 컬럼 정의 (동적 추가/삭제) | 영문으로 시작, `[a-zA-Z][a-zA-Z0-9_]*` |
| **MCP 활성화** | MCP 도구에서 접근 허용 여부 | 체크박스 |

### 필드 정의

각 필드에 다음을 설정할 수 있습니다:

| 속성 | 설명 |
|------|------|
| **이름** | 필드명 |
| **타입** | String, Number, Boolean, Date, Array, Object, Mixed |
| **필수** | 필수 필드 여부 |
| **유니크** | 고유값 여부 |

### 시스템 필드

다이얼로그에 읽기 전용으로 표시됩니다:

- `_id` (String) — 고유 식별자
- `createdBy` (String) — 생성자 ID
- `createdAt` (Date) — 생성 일시
- `updatedAt` (Date) — 수정 일시

## 테이블 상세 화면

테이블 목록에서 테이블을 클릭하면 상세 화면으로 이동합니다.

### 탭 구조

| 탭 | 설명 |
|-----|------|
| **Data** | 데이터 조회, 필터링, 페이지네이션 |
| **Schema** | 필드 목록, 필드 추가/수정/삭제, 버전 관리 |
| **Indexes** | 인덱스 목록, 인덱스 생성/삭제, 버전 관리 |
| **Permissions** | 그룹별 CRUD 권한 설정 |
| **API Docs** | API 사용 예시, cURL 예제 |

### Data 탭

- 테이블 데이터를 표 형태로 표시합니다
- 필터, 정렬, 페이지네이션을 지원합니다
- 행을 클릭하여 상세 정보를 확인할 수 있습니다

### Schema 탭

- 현재 스키마의 필드 목록 (이름, 타입, 필수, 유니크)을 표시합니다
- 필드를 추가, 수정, 삭제할 수 있습니다
- 스키마 버전 이력을 확인하고 이전 버전으로 롤백할 수 있습니다

### Indexes 탭

- 현재 인덱스 목록 (이름, 필드, 유니크, 스파스)을 표시합니다
- 인덱스를 생성하거나 삭제할 수 있습니다
- 인덱스 버전 이력을 확인하고 롤백할 수 있습니다

### Permissions 탭

- 그룹별(admin, user, self, guest) CRUD 권한 매트릭스를 표시합니다
- 각 그룹의 Create, Read, Update, Delete 권한을 개별적으로 설정할 수 있습니다

# 모니터링 대시보드

> 프로젝트와 환경의 상태를 모니터링하는 방법을 안내합니다.

## 개요

bkend는 Organization, Project, Environment 단위로 대시보드를 제공합니다. 각 대시보드에서 리소스 사용량과 서비스 상태를 확인할 수 있습니다.

---

## Organization 대시보드

Organization 대시보드는 조직 전체의 요약 정보를 표시합니다.

### 주요 통계

| 메트릭 | 설명 |
|--------|------|
| **프로젝트 수** | 조직 내 총 프로젝트 수 |
| **멤버 수** | 조직 내 총 팀 멤버 수 |
| **저장소 사용량** | 전체 파일 저장소 사용량 |
| **테이블 수** | 전체 테이블 수 |

### 추가 정보

- **최근 프로젝트** — 최근에 생성되거나 활동이 있는 프로젝트 목록
- **최근 활동** — 조직 내 최근 작업 이력

---

## Project 대시보드

프로젝트 단위의 통계를 제공합니다.

| 메트릭 | 설명 |
|--------|------|
| **테이블 수** | 프로젝트 내 총 테이블 수 |
| **User 수** | 프로젝트의 총 User 수 |
| **환경 수** | 활성 환경 수 |
| **저장소 사용량** | 프로젝트의 파일 저장소 사용량 |

---

## Environment 대시보드

개별 환경의 상세 통계를 제공합니다.

| 메트릭 | 설명 |
|--------|------|
| **테이블 수** | 환경 내 테이블 수 |
| **레코드 수** | 환경 내 총 데이터 레코드 수 |
| **User 수** | 환경에 등록된 User 수 |
| **저장소 사용량** | 환경의 파일 저장소 사용량 |

---

## 모니터링 기준

서비스 상태를 판단하는 기준입니다.

| 메트릭 | 정상 | 주의 | 위험 |
|--------|------|------|------|
| **에러율** | 0~1% | 1~5% | 5% 초과 |
| **응답 시간** | 0~200ms | 200~500ms | 500ms 초과 |
| **메모리 사용률** | 0~70% | 70~85% | 85% 초과 |
| **DB 연결 수** | 0~80 | 80~100 | 100 초과 |

---

## 활동 로그 확인하기

프로젝트의 **Activities** 페이지에서 최근 활동 내역을 확인할 수 있습니다.

### 표시 정보

각 활동 항목은 다음 정보를 포함합니다.

| 항목 | 설명 |
|------|------|
| **이벤트 타입** | 어떤 리소스에 대한 이벤트인지 |
| **액션** | created, updated, deleted, deployed 등 |
| **메시지** | 작업 설명 |
| **엔티티 배지** | 리소스 유형 표시 |
| **시간** | 상대 시간으로 표시 (예: 5분 전, 2시간 전) |

### 필터링

| 필터 | 옵션 |
|------|------|
| **검색** | 메시지 텍스트로 검색 |
| **엔티티 타입** | table, environment, project, invitation |
| **액션** | created, updated, deleted, deployed |

### 액션 색상 구분

| 액션 | 색상 | 의미 |
|------|------|------|
| `created` | 초록색 | 리소스 생성 |
| `updated` | 파란색 | 리소스 수정 |
| `deleted` | 빨간색 | 리소스 삭제 |
| `deployed` | 보라색 | 환경 배포 |
| `accepted` | 초록색 | 초대 수락 |
| `revoked` | 노란색 | 초대 취소 |

> 💡 **Tip** - 새로고침 버튼을 클릭하면 최신 활동을 즉시 확인할 수 있습니다. 페이지당 20개 항목이 표시되며, 페이지네이션으로 이전 활동을 조회할 수 있습니다.

## 일일 점검 체크리스트

프로덕션 환경을 운영 중이라면 다음 항목을 정기적으로 점검하세요.

- [ ] 모든 환경 상태가 `active`인지 확인
- [ ] 에러율이 1% 미만인지 확인
- [ ] 응답 시간이 500ms 미만인지 확인
- [ ] 저장소 사용량이 한도의 80% 미만인지 확인
- [ ] 비정상적인 활동 로그가 없는지 확인

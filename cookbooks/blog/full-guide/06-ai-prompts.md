# AI 프롬프트 모음

{% hint style="info" %}
💡 블로그 프로젝트에서 AI에게 요청할 수 있는 프롬프트를 시나리오별로 정리했습니다.
{% endhint %}

## 개요

이 문서는 블로그 프로젝트의 각 기능을 AI에게 자연어로 요청하는 프롬프트 레퍼런스입니다. MCP 도구가 연결된 상태에서 아래 프롬프트를 그대로 사용하거나, 상황에 맞게 변형하여 사용하세요.

### 선행 조건

| 필요 항목 | 설명 | 참조 |
|----------|------|------|
| MCP 도구 연결 | AI 도구에서 bkend MCP 서버 연결 완료 | [AI 도구 연동](../../../ko/ai-tools/01-overview.md) |
| 프로젝트 선택 | MCP 세션에서 블로그 프로젝트 선택 | [AI 도구 연동](../../../ko/ai-tools/01-overview.md) |

## 테이블 관리

프로젝트의 테이블을 생성하고 스키마를 관리하는 프롬프트입니다.

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"블로그 글을 저장하고 싶어요. 제목, 본문, 대표 이미지, 카테고리, 공개 여부를 관리할 수 있게 해주세요. 만들기 전에 어떤 구조로 만들지 먼저 보여주세요."
{% endhint %}

{% hint style="info" %}
💡 AI가 아래와 비슷한 구조를 제안하는지 확인하세요.

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| title | 게시글 제목 | "나의 첫 블로그" |
| content | 본문 내용 | "안녕하세요..." |
| coverImage | 대표 이미지 URL | (업로드 후 연결) |
| category | 카테고리 | "여행" |
| isPublished | 공개 여부 | `true` / `false` |
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"게시글 저장 구조가 어떻게 되어 있는지 확인해주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"블로그 글에 태그를 붙이고 싶어요. 태그 이름과 URL용 식별자를 관리할 수 있게 해주세요. 같은 이름의 태그는 중복으로 만들 수 없어야 해요. 그리고 게시글에도 태그를 여러 개 연결할 수 있게 해주세요. 만들기 전에 어떤 구조로 만들지 먼저 보여주세요."
{% endhint %}

{% hint style="info" %}
💡 AI가 아래와 비슷한 구조를 제안하는지 확인하세요.

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| name | 태그 이름 | "여행" |
| slug | URL용 식별자 | "travel" |
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"사용자가 관심 있는 글을 북마크할 수 있게 하고 싶어요. 어떤 글을 북마크했는지 저장할 수 있게 해주세요. 만들기 전에 어떤 구조로 만들지 먼저 보여주세요."
{% endhint %}

{% hint style="info" %}
💡 AI가 아래와 비슷한 구조를 제안하는지 확인하세요.

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| articleId | 북마크한 글 | (게시글 ID) |
{% endhint %}

{% hint style="info" %}
💡 테이블 생성은 MCP 도구로만 수행할 수 있습니다. REST API로는 테이블을 생성할 수 없습니다.
{% endhint %}

## 게시글 관리

게시글 작성, 조회, 수정, 삭제를 AI에게 요청하는 프롬프트입니다.

### 게시글 작성

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"블로그에 새 글을 작성해주세요. 제목은 '제주도 3박4일 여행기', 카테고리는 '여행'으로요. 아직 공개하지 말고 초안으로 저장해주세요."
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"기술 카테고리로 글을 작성하고 바로 공개해주세요. 제목은 'React 19 새로운 기능 정리', 내용은 'React 19에서 추가된 주요 기능들을 소개합니다'로요."
{% endhint %}

### 게시글 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"방금 작성한 글 내용을 보여주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"여행 카테고리 글 중에서 최신순으로 5개만 보여주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"공개된 글만 최신순으로 보여주세요"
{% endhint %}

### 게시글 수정

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"방금 작성한 여행 글 제목을 '제주도 3박4일 여행기 - 완벽 가이드'로 바꿔주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"이 글을 공개해주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"이 글의 카테고리를 '일상'에서 '여행'으로 변경해주세요"
{% endhint %}

### 게시글 삭제

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"'테스트 글'을 삭제해주세요"
{% endhint %}

{% hint style="warning" %}
⚠️ 삭제된 글은 복구할 수 없습니다. AI가 삭제 전 확인을 요청하면 내용을 확인한 후 진행하세요.
{% endhint %}

## 이미지 관리

{% hint style="warning" %}
⚠️ 파일 업로드는 클라이언트에서 REST API로 직접 수행합니다. MCP 도구로는 파일을 업로드할 수 없습니다. 업로드 후 이미지 URL을 게시글에 연결하는 작업은 MCP로 요청할 수 있습니다.
{% endhint %}

### 이미지 연결

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"제주도 여행 글의 대표 이미지를 이 URL로 설정해주세요: https://cdn.example.com/cover-jeju.jpg"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"이 글의 대표 이미지를 제거해주세요"
{% endhint %}

### 파일 정보 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"업로드한 이미지 정보를 확인해주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"방금 확인한 이미지 파일을 삭제해주세요"
{% endhint %}

## 태그 관리

태그를 생성하고 게시글에 연결하는 프롬프트입니다.

### 태그 생성

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"블로그에서 사용할 태그를 만들어주세요: 여행, 맛집, 기술, 일상"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"'프론트엔드'라는 태그를 만들어주세요. URL에서 쓸 이름은 'frontend'로 해주세요."
{% endhint %}

### 태그 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"현재 만들어진 태그 목록을 보여주세요"
{% endhint %}

### 게시글에 태그 연결

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"제주도 여행 글에 '여행', '맛집' 태그를 달아주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"이 글에 달린 태그를 전부 제거해주세요"
{% endhint %}

### 태그별 게시글 필터링

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"'여행' 태그가 달린 글을 보여주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"'여행'과 '맛집' 태그가 모두 달린 글을 보여주세요"
{% endhint %}

### 태그 수정 및 삭제

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"'여행' 태그 이름을 '해외여행'으로 바꿔주세요"
{% endhint %}

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"'일상' 태그를 삭제해주세요"
{% endhint %}

{% hint style="warning" %}
⚠️ 태그를 삭제해도 글에 남아있는 태그 연결은 자동으로 제거되지 않습니다. 필요 시 관련 글의 태그도 함께 정리해달라고 요청하세요.
{% endhint %}

## 복합 시나리오

여러 기능을 조합한 고급 프롬프트입니다. AI가 여러 MCP 도구를 순차적으로 호출하여 처리합니다.

### 이미지 포함 글 작성 + 태그 추가

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"여행 카테고리로 새 글을 작성해주세요. 제목은 '서울 카페 투어'이고, 대표 이미지는 https://cdn.example.com/seoul-cafe.jpg 로 설정해주세요. '여행', '서울', '카페' 태그도 달아주세요."
{% endhint %}

AI가 순차적으로 처리합니다:

1. 글 생성
2. 대표 이미지 설정 + 태그 연결

### 콘텐츠 기반 태그 추천

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"방금 작성한 글 내용을 분석해서 어울리는 태그를 추천해주세요. 기존에 만들어둔 태그 중에서 골라주세요."
{% endhint %}

AI가 순차적으로 처리합니다:

1. 글 내용 조회
2. 기존 태그 목록 조회
3. 내용과 태그를 매칭하여 추천

### 초안 글 일괄 정리

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"아직 공개하지 않은 글 목록을 보여주세요. 각 글의 제목과 작성일을 알려주세요."
{% endhint %}

### 카테고리별 글 현황

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"카테고리별로 글이 몇 개인지 정리해주세요. 여행, 기술, 일상, 맛집 카테고리별로 공개/비공개 구분해서 알려주세요."
{% endhint %}

AI가 각 카테고리별로 글 수를 조회하여 결과를 종합합니다.

### 블로그 현황 요약

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**
"블로그 현황을 요약해주세요. 전체 글 수, 공개/비공개 수, 태그 수, 북마크 수를 알려주세요."
{% endhint %}

AI가 여러 데이터를 조회하여 종합합니다.

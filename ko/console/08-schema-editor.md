# 스키마 편집기

{% hint style="info" %}
💡 스키마 편집기에서 테이블의 필드를 추가, 수정, 삭제하세요.
{% endhint %}

## 개요

스키마 편집기는 테이블의 필드 구조를 관리하는 도구입니다. 필드 타입, 필수 여부를 설정하고, 스키마 버전 히스토리를 통해 변경 이력을 추적합니다.

***

## 스키마 편집기 열기

1. **데이터베이스** 페이지에서 테이블을 클릭하세요.
2. **Schema** 탭을 클릭하세요.

![스키마 편집기](../.gitbook/assets/2026-02/console-schema-editor.png)

***

## 필드 목록

스키마 탭에서는 현재 테이블의 모든 필드를 테이블 형태로 확인할 수 있습니다.

| 컬럼 | 설명 |
|------|------|
| 필드명 | 필드 이름 (타입별 아이콘 포함) |
| 타입 | String, Number, Boolean, Date, Array, Object, Mixed |
| 필수 | 필수 여부 |
| 액션 | 수정, 삭제 버튼 |

### 필드 타입

| 타입 | 설명 |
|------|------|
| `String` | 문자열 |
| `Number` | 숫자 (정수, 소수) |
| `Boolean` | true / false |
| `Date` | 날짜/시간 |
| `Array` | 배열 |
| `Object` | 중첩 객체 |
| `Mixed` | 혼합 타입 (제한 없음) |

{% hint style="warning" %}
⚠️ 시스템 필드(`_id`, `createdBy`, `createdAt`, `updatedAt`)는 수정하거나 삭제할 수 없습니다.
{% endhint %}

***

## 필드 추가하기

1. **Add Field** 버튼을 클릭하세요.
2. 다음 정보를 입력하세요.

| 항목 | 필수 | 설명 |
|------|:----:|------|
| **필드명** | ✅ | 영문자로 시작, 영문/숫자/언더스코어 허용 |
| **타입** | ✅ | 드롭다운에서 선택 |
| **Required** | - | 필수 여부 체크 |

3. **저장**을 클릭하세요.

<!-- 📸 IMG: 필드 추가 모달 -->
![컬럼 추가](../.gitbook/assets/2026-02/console-column-add.png)

***

## 필드 수정하기

1. 수정할 필드의 **편집** 버튼을 클릭하세요.
2. 타입, 필수 여부를 변경하세요.
3. **저장**을 클릭하세요.

***

## 필드 삭제하기

{% hint style="danger" %}
🚨 **위험** — 필드를 삭제하면 해당 필드의 데이터가 손실될 수 있습니다.
{% endhint %}

1. 삭제할 필드의 **삭제** 버튼을 클릭하세요.
2. 확인 대화상자에서 **삭제**를 클릭하세요.

***

## 스키마 버전 히스토리

스키마를 변경할 때마다 새 버전이 생성됩니다. **History** 버튼을 클릭하면 모든 버전의 변경 이력을 확인할 수 있습니다.

| 컬럼 | 설명 |
|------|------|
| 버전 | v1, v2, v3... |
| 상태 | 현재 / 이전 |
| 생성일 | 버전 생성 날짜 |
| 생성자 | 변경한 사용자 |
| 액션 | 스키마 보기, 롤백 |

### 스키마 롤백하기

{% hint style="danger" %}
🚨 **위험** — 롤백은 되돌릴 수 없습니다. 현재 스키마가 선택한 버전으로 교체됩니다.
{% endhint %}

1. **History** 버튼을 클릭하세요.
2. 원하는 버전의 **Rollback** 버튼을 클릭하세요.
3. 확인 대화상자에서 **롤백**을 클릭하세요.

{% hint style="info" %}
💡 AI 도구에서도 필드를 관리할 수 있습니다. 자세한 내용은 [Database MCP 도구](../ai-tools/12-mcp-db-tools.md)를 참고하세요.
{% endhint %}

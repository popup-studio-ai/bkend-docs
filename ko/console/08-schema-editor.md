# 스키마 편집기

{% hint style="info" %}
💡 테이블의 컬럼을 추가, 수정, 삭제하는 스키마 편집기 사용법을 안내합니다.
{% endhint %}

## 개요

스키마 편집기에서 테이블의 컬럼 구조를 설계합니다. 7가지 컬럼 타입을 지원하며, 필수 여부, 기본값, 고유값 제약 조건을 설정할 수 있습니다.

***

## 스키마 편집기 열기

1. **데이터베이스** → 테이블을 클릭하세요.
2. **스키마** 탭을 클릭하세요.

***

## 컬럼 타입

| 타입 | 설명 | 예시 |
|------|------|------|
| **string** | 문자열 | 이름, 이메일, URL |
| **number** | 숫자 (정수/실수) | 나이, 가격, 수량 |
| **boolean** | 참/거짓 | 공개 여부, 활성 상태 |
| **date** | 날짜/시간 | 생일, 예약일 |
| **object** | JSON 객체 | 메타데이터, 설정 |
| **array** | JSON 배열 | 태그 목록, 카테고리 |
| **reference** | 다른 테이블 참조 | 작성자 ID, 카테고리 ID |

***

## 컬럼 추가하기

{% tabs %}
{% tab title="콘솔" %}
1. **필드 추가** 버튼을 클릭하세요.
2. 다음 정보를 입력하세요.

| 필드 | 설명 |
|------|------|
| **컬럼 이름** | 필드 식별자 (snake_case 권장) |
| **타입** | 7가지 타입 중 선택 |
| **필수** | NOT NULL 제약 조건 |
| **기본값** | 값이 없을 때 자동 적용 |
| **고유** | UNIQUE 제약 조건 |

3. **저장**을 클릭하세요.
{% endtab %}
{% tab title="MCP" %}
AI 도구에서 자연어로 요청하세요.

```text
"posts 테이블에 views 컬럼을 추가해줘. 숫자 타입, 기본값 0"
```
{% endtab %}
{% endtabs %}

***

## 컬럼 수정하기

1. 수정할 컬럼의 **편집** 아이콘을 클릭하세요.
2. 타입, 필수 여부, 기본값 등을 변경하세요.
3. **저장**을 클릭하세요.

{% hint style="warning" %}
⚠️ 데이터가 있는 컬럼의 타입을 변경하면 기존 데이터와 호환되지 않을 수 있습니다. 신중하게 변경하세요.
{% endhint %}

***

## 컬럼 삭제하기

{% hint style="danger" %}
🚨 **위험** — 컬럼을 삭제하면 해당 컬럼의 모든 데이터가 영구 삭제됩니다.
{% endhint %}

1. 삭제할 컬럼의 **삭제** 아이콘을 클릭하세요.
2. 확인 후 컬럼이 삭제됩니다.

***

## 자동 생성 필드

모든 테이블에는 다음 필드가 자동으로 포함됩니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 식별자 (자동 생성) |
| `createdAt` | date | 생성 일시 (자동 기록) |
| `updatedAt` | date | 수정 일시 (자동 갱신) |

# 프로젝트 설정

{% hint style="info" %}
💡 프로젝트 이름 변경, 정보 확인, 삭제 등의 설정을 관리하세요.
{% endhint %}

## 개요

프로젝트 설정 페이지에서 프로젝트의 기본 정보를 확인하고, 이름이나 설명을 변경할 수 있습니다.

***

## 설정 페이지 열기

1. 프로젝트를 선택하세요.
2. 사이드바에서 **설정**을 클릭하세요.

<!-- 📸 IMG: 프로젝트 설정 화면 -->
![프로젝트 설정](../.gitbook/assets/2026-02/console-project-settings.png)

***

## 프로젝트 정보

설정 페이지 상단에서 프로젝트의 기본 정보를 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| **Project ID** | 프로젝트 고유 식별자 (복사 가능) |
| **Slug** | URL에 사용되는 식별자 |
| **생성 날짜** | 프로젝트가 생성된 날짜 |
| **클라우드** | 클라우드 제공자 (예: AWS) |
| **리전** | 데이터 저장 위치 (예: Seoul) |

{% hint style="info" %}
💡 **Project ID** 옆의 복사 버튼을 클릭하면 ID를 클립보드에 복사할 수 있습니다. REST API 호출 시 `X-Project-Id` 헤더에 이 값을 사용하세요.
{% endhint %}

***

## 프로젝트 정보 수정하기

1. **일반 설정** 카드에서 다음 필드를 수정하세요.

| 필드 | 필수 | 설명 |
|------|:----:|------|
| **프로젝트 이름** | ✅ | 1~100자 |
| **설명** | - | 최대 500자 |

2. **저장**을 클릭하세요.

{% hint style="warning" %}
⚠️ Owner 또는 Admin 역할만 프로젝트 설정을 변경할 수 있습니다.
{% endhint %}

***

## 프로젝트 삭제하기

{% hint style="danger" %}
🚨 **위험** — 프로젝트를 삭제하면 모든 환경, 테이블, 데이터, 파일, User가 영구적으로 삭제됩니다. 이 작업은 되돌릴 수 없습니다.
{% endhint %}

1. 설정 페이지 하단의 **위험 영역**에서 **프로젝트 삭제** 버튼을 클릭하세요.
2. 프로젝트 이름을 입력하여 확인하세요.
3. **삭제**를 클릭하세요.

삭제가 완료되면 조직의 프로젝트 목록으로 자동 이동됩니다.

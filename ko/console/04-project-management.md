# 프로젝트 관리

{% hint style="info" %}
💡 Project는 하나의 서비스 또는 앱에 대응합니다. 프로젝트를 생성하고 관리하는 방법을 안내합니다.
{% endhint %}

## 개요

Project는 Organization 하위에 생성됩니다. 프로젝트를 만들면 `dev` 환경이 자동 프로비저닝되며, 테이블 설계, 인증 설정, API Key 발급 등 모든 기능을 사용할 수 있습니다.

***

## 프로젝트 생성하기

1. 사이드바에서 **프로젝트**를 클릭하세요.
2. **프로젝트 생성** 버튼을 클릭하세요.
3. 다음 정보를 입력하세요.

| 필드 | 설명 | 예시 |
|------|------|------|
| **프로젝트 이름** | 프로젝트 표시 이름 | My Blog |
| **프로젝트 Slug** | URL 식별자 (자동 생성 가능) | my-blog |
| **리전** | 데이터 저장 리전 | Seoul (ap-northeast-2) |
| **클라우드** | 클라우드 제공자 | AWS |

4. **생성**을 클릭하세요.

{% hint style="warning" %}
⚠️ 프로젝트를 생성하면 `dev` 환경이 자동 프로비저닝됩니다. **Active** 상태가 될 때까지 약 30초를 기다리세요.
{% endhint %}

***

## 프로젝트 목록 확인하기

사이드바에서 **프로젝트**를 클릭하면 현재 Organization의 모든 프로젝트를 확인할 수 있습니다.

| 표시 정보 | 설명 |
|----------|------|
| **프로젝트 이름** | 프로젝트 표시 이름 |
| **Slug** | URL 식별자 |
| **리전** | 데이터 저장 위치 |
| **환경 수** | 생성된 환경 개수 |
| **상태** | Active / Provisioning |

***

## 프로젝트 선택하기

프로젝트 목록에서 프로젝트를 클릭하면 **프로젝트 레벨 사이드바**로 전환됩니다. 환경 관리, 데이터베이스, 인증, 스토리지 등 프로젝트별 기능에 접근할 수 있습니다.

***

## 프로젝트 설정 변경하기

프로젝트 레벨 사이드바에서 **프로젝트 설정**을 클릭하여 프로젝트 정보를 수정하세요.

{% hint style="info" %}
💡 Project ID는 프로젝트 설정 페이지에서 확인할 수 있습니다. API Key에 프로젝트 정보가 포함되어 있으므로, 별도의 헤더 없이 API Key만으로 프로젝트를 식별합니다.
{% endhint %}

***

## 프로젝트 삭제하기

{% hint style="danger" %}
🚨 **위험** — 프로젝트를 삭제하면 모든 환경, 테이블, User, 파일이 영구 삭제됩니다. 이 작업은 되돌릴 수 없습니다.
{% endhint %}

1. 프로젝트 레벨 사이드바에서 **프로젝트 설정**을 클릭하세요.
2. 페이지 하단의 **프로젝트 삭제** 섹션에서 **삭제**를 클릭하세요.
3. 프로젝트 이름을 입력하고 **삭제 확인**을 클릭하세요.

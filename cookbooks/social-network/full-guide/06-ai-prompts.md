# 06. AI 프롬프트 모음

{% hint style="info" %}
💡 소셜 네트워크 프로젝트의 각 기능을 AI에게 자연어로 요청하는 프롬프트 레퍼런스입니다. MCP 연결이 완료된 상태에서 아래 프롬프트를 그대로 사용하세요.
{% endhint %}

## 이 장에서 다루는 내용

- 테이블 생성/스키마 관리 프롬프트
- 프로필 CRUD 프롬프트
- 게시물, 댓글, 좋아요 프롬프트
- 팔로우/언팔로우 프롬프트
- 피드 조회 및 필터링 프롬프트
- 복합 시나리오 프롬프트

***

## 기본 설계 참조

AI에게 테이블 생성을 요청하면, 아래와 비슷한 구조를 제안하는지 확인하세요.

### profiles (사용자 프로필)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| nickname | 닉네임 | "김코딩" |
| bio | 자기소개 | "개발을 좋아합니다" |
| avatarUrl | 프로필 사진 URL | (업로드 후 연결) |
| userId | 사용자 식별값 | (사용자 ID) |

### posts (게시물)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| content | 게시물 내용 | "오늘 날씨가 좋네요" |
| imageUrl | 첨부 이미지 URL | (업로드 후 연결) |
| likesCount | 좋아요 수 | 0 |
| commentsCount | 댓글 수 | 0 |

### comments (댓글)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| postId | 댓글 달 게시물 | (게시물 ID) |
| content | 댓글 내용 | "저도 그렇게 생각해요" |

### likes (좋아요)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| postId | 좋아요 한 게시물 | (게시물 ID) |

### follows (팔로우 관계)

| 필드 | 설명 | 예시 값 |
|------|------|---------|
| followerId | 팔로우 하는 사람 | (사용자 ID) |
| followingId | 팔로우 받는 사람 | (사용자 ID) |

## 테이블 관리

데이터를 저장할 곳을 만들고 관리하는 프롬프트입니다.

### 전체 테이블 한 번에 만들기

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"소셜 네트워크를 만들고 싶어요. 사용자 프로필, 게시물, 댓글, 좋아요, 팔로우 기능이 필요해요. 필요한 저장소를 전부 만들어주세요. 만들기 전에 어떤 구조로 만들지 먼저 보여주세요."
{% endhint %}

### 구조 확인

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"게시물 저장소의 구조를 확인해주세요."
{% endhint %}

### 항목 추가

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프로필에 개인 웹사이트 주소도 저장할 수 있게 해주세요."
{% endhint %}

## 프로필 관리

프로필 생성, 수정, 사진 업로드에 사용하는 프롬프트입니다.

### 프로필 생성

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"내 프로필을 만들어주세요. 닉네임은 '개발자김', 자기소개는 '풀스택 개발자입니다'로 해주세요."
{% endhint %}

### 프로필 수정

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"내 자기소개를 '백엔드 개발에 관심이 많습니다'로 바꿔주세요."
{% endhint %}

### 프로필 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"'김소셜'님의 프로필 정보를 보여주세요."
{% endhint %}

### 프로필 사진 변경

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"프로필 사진을 바꾸고 싶어요. 이미지 업로드 준비를 해주세요."
{% endhint %}

{% hint style="info" %}
💡 AI가 업로드용 URL을 발급하면, 앱에서 해당 URL로 이미지를 업로드한 뒤 프로필 사진이 자동 연결됩니다.
{% endhint %}

### 게시물 삭제

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"방금 그 게시물에 '축하합니다! 화이팅!' 댓글을 달아주세요."
{% endhint %}

### 댓글 목록 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

### 이미지 첨부 게시물

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"이미지를 첨부해서 게시물을 올리고 싶어요. 이미지 업로드 준비를 해주세요."
{% endhint %}

{% hint style="info" %}
💡 이미지 첨부 게시물은 3단계로 진행됩니다: (1) 업로드 URL 발급 → (2) 이미지 업로드 → (3) 이미지를 포함하여 게시물 생성. AI가 단계별로 안내합니다.
{% endhint %}

## 복합 시나리오

여러 작업을 한 번에 요청하는 프롬프트입니다. AI가 필요한 단계를 자동으로 분석하고 순차적으로 처리합니다.

### 신규 계정 초기 설정

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"방금 가입했어요. 프로필을 설정하고 첫 게시물도 올려주세요. 닉네임은 '소셜러버', 자기소개는 '소통을 좋아합니다', 첫 게시물은 '안녕하세요! 처음 가입했습니다'로 해주세요."
{% endhint %}

### 활동 요약 확인

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"내 활동 요약을 보여주세요. 게시물이 몇 개인지, 팔로워가 몇 명인지, 내가 팔로우하는 사람이 몇 명인지 전부 알려주세요."
{% endhint %}

AI 응답 예시:

```text
활동 요약:
- 게시물: 15개
- 팔로워: 42명
- 팔로잉: 28명
- 가장 인기 게시물: "AI 트렌드 분석" (좋아요 45개)
```

### 특정 사용자 정보 종합 조회

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"'김소셜'님의 프로필과 최근 게시물 5개, 팔로워 수를 알려주세요."
{% endhint %}

### 게시물 + 댓글 + 좋아요 한 번에

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"'주말 프로젝트 완성!' 게시물을 올리고, '김소셜'님의 최신 게시물에 '멋져요!' 댓글과 좋아요를 남겨주세요."
{% endhint %}

### 피드 큐레이션

{% hint style="success" %}
✅ **AI에게 이렇게 말해보세요**

"이번 주 인기 게시물 중에서 내가 팔로우하는 사람의 게시물만 골라서 보여주세요."
{% endhint %}

## 프롬프트 작성 팁

{% hint style="info" %}
💡 효과적인 프롬프트를 작성하려면 다음을 참고하세요.

- **구체적인 내용을 포함하세요**: "게시물 올려주세요" 보다 "'안녕하세요!' 게시물을 올려주세요"가 정확합니다.
- **여러 작업을 한 번에 요청할 수 있습니다**: AI가 필요한 단계를 자동으로 분리하여 처리합니다.
- **조건을 자연어로 설명하세요**: "오늘 올라온 게시물 중 좋아요 10개 이상만 보여주세요"처럼 원하는 조건을 구체적으로 말하세요.
- **새로운 저장소를 만들 때는 "먼저 보여주세요"를 붙이세요**: AI가 만들기 전에 구조를 보여주고 확인을 받습니다.
{% endhint %}

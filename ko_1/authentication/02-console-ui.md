# 콘솔에서 Auth 관리하기

> bkend 콘솔의 Authentication 화면에서 인증 설정과 User를 관리하는 방법을 안내합니다.

## 개요

콘솔의 **Auth** 메뉴에서 인증 방식 설정, User 목록 조회, 소셜 로그인 설정, 세션 관리를 수행할 수 있습니다.

---

## Auth 페이지 접근하기

1. 콘솔에서 프로젝트를 선택하세요.
2. Sidebar의 **Auth** 메뉴를 클릭하세요.
3. 상단의 환경 탭에서 원하는 환경(dev, staging, prod)을 선택하세요.

---

## 화면 구성

### User 목록

| 영역 | 설명 |
|------|------|
| **환경 탭** | dev, staging, prod 환경 전환 |
| **User 목록** | 가입한 User 목록 (이름, 이메일, 가입일, 상태) |
| **검색** | 이메일, 이름으로 User 검색 |
| **필터** | 인증 방식(이메일, 소셜), 인증 상태 필터 |

### 인증 설정

| 설정 | 설명 |
|------|------|
| **이메일 인증** | 회원가입 시 이메일 인증 필수 여부 |
| **비밀번호 정책** | 비밀번호 최소 길이, 복잡도 설정 |
| **소셜 로그인** | OAuth 제공자별 Client ID, Secret 설정 |
| **2FA** | 2단계 인증 활성화 설정 |

---

## 소셜 로그인 설정하기

1. Auth 페이지에서 **소셜 로그인** 설정 영역으로 이동하세요.
2. 원하는 제공자(Google, GitHub)를 선택하세요.
3. **Client ID**와 **Client Secret**을 입력하세요.
4. **Redirect URI**를 확인하고 OAuth 제공자의 설정에 추가하세요.
5. **활성화** 토글을 켜세요.

---

## User 상세 보기

User 목록에서 User를 클릭하면 상세 정보를 확인할 수 있습니다:

| 항목 | 설명 |
|------|------|
| **프로필** | 이름, 이메일, 프로필 이미지, 가입일 |
| **인증 방식** | 이메일, 연동된 소셜 계정 목록 |
| **세션** | 활성 세션 목록 (디바이스, IP, 마지막 활동) |
| **활동 로그** | 로그인/로그아웃 이력 |

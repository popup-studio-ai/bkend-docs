# 02. Implementing the Product Catalog

{% hint style="info" %}
💡 Create the products table, then implement product registration/browsing/editing/deletion and image upload.
{% endhint %}

## Overview

In this chapter, you will implement the product catalog — the core of the shopping mall.

- Create the `products` table
- Register products (name, description, price, category, stock)
- Upload product images (Presigned URL)
- List products (category filter, price range, sorting)
- View product details
- Edit/delete products
- Inventory management

## Step 1: Create the products Table

Create the `products` table to store product data.

### Table Schema

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `name` | String | ✅ | Product name |
| `description` | String | ✅ | Product description |
| `price` | Number | ✅ | Price |
| `category` | String | ✅ | Category (e.g., Clothing, Electronics, Food) |
| `stock` | Number | ✅ | Stock quantity |
| `imageUrl` | String | - | Product image URL |
| `isActive` | Boolean | - | Whether on sale (default: true) |

{% hint style="info" %}
💡 `id`, `createdBy`, `createdAt`, and `updatedAt` are system fields automatically generated by bkend. You don't need to provide them.
{% endhint %}

{% hint style="success" %}
✅ **Try saying this to AI**

"I want to manage products for a shopping mall. I need to store product name, description, price, category, stock quantity, product image, and whether it's on sale. Show me the structure before creating it."
{% endhint %}

{% hint style="info" %}
💡 Check that the AI suggests a structure similar to the one below.
{% endhint %}

| Field | Description | Example Value |
|-------|-------------|---------------|
| name | Product name | "Premium Cotton T-Shirt" |
| description | Product description | "Soft 100% cotton..." |
| price | Price | 29000 |
| category | Category | "Clothing" |
| stock | Stock quantity | 100 |
| imageUrl | Product image URL | (linked after upload) |
| isActive | On sale or not | true / false |
{% endtab %}

{% tab title="Console" %}
1. Sign in to the console.
2. Go to the **Tables** menu.
3. Click **Add New Table**.
4. Enter `products` as the table name.
5. Add the fields as described in the schema above.
6. Click **Save** to create the table.

## Step 2: Register a Product

Once the table is created, register your first product.

{% tabs %}
{% tab title="MCP (AI Tools)" %}
{% hint style="success" %}
✅ **Try saying this to AI**

"Register a new product. Premium Cotton T-Shirt, a soft 100% cotton basic t-shirt. Price 29,000 won, category is Clothing, stock 100."
{% endhint %}

The AI registers the product and reports the result.
{% endtab %}

{% tab title="Console + REST API" %}
```bash
curl -X POST https://api-client.bkend.ai/v1/data/products \
  -H "Content-Type: application/json" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}" \
  -d '{
    "name": "Premium Cotton T-Shirt",
    "description": "A soft basic t-shirt made of 100% cotton.",
    "price": 29000,
    "category": "Clothing",
    "stock": 100,
    "isActive": true
  }'
```

**bkendFetch example:**

```javascript
const product = await bkendFetch('/v1/data/products', {
  method: 'POST',
  body: {
    name: 'Premium Cotton T-Shirt',
    description: 'A soft basic t-shirt made of 100% cotton.',
    price: 29000,
    category: 'Clothing',
    stock: 100,
    isActive: true,
  },
});

console.log('Registered product:', product);
```

**Response example:**

```json
{
  "id": "product_abc123",
  "name": "Premium Cotton T-Shirt",
  "description": "A soft basic t-shirt made of 100% cotton.",
  "price": 29000,
  "category": "Clothing",
  "stock": 100,
  "isActive": true,
  "createdBy": "user_abc123",
  "createdAt": "2025-01-15T10:00:00Z"
}
```

## Step 3: Upload Product Image

To add an image to a product, get a Presigned URL, upload the file, then link the returned URL to the product.

{% tabs %}
{% tab title="MCP (AI Tools)" %}
{% hint style="success" %}
✅ **Try saying this to AI**

"I want to add a product image to the Premium Cotton T-Shirt. Upload the image file."
{% endhint %}

The AI uploads the image and automatically links it to the product.
{% endtab %}

{% tab title="Console + REST API" %}

### 3-1. Get Presigned URL

```bash
curl -X POST https://api-client.bkend.ai/v1/files/presigned-url \
  -H "Content-Type: application/json" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}" \
  -d '{
    "filename": "premium-tshirt.jpg",
    "contentType": "image/jpeg"
  }'
```

**Response example:**

```json
{
  "url": "https://storage.bkend.ai/...",
  "key": "{file_key}",
  "filename": "premium-tshirt.jpg",
  "contentType": "image/jpeg"
}
```

### 3-2. Upload File

```bash
curl -X PUT "{url}" \
  -H "Content-Type: image/jpeg" \
  --data-binary @premium-tshirt.jpg
```

**bkendFetch example:**

```javascript
// 1. Get Presigned URL
const presigned = await bkendFetch('/v1/files/presigned-url', {
  method: 'POST',
  body: {
    filename: 'premium-tshirt.jpg',
    contentType: 'image/jpeg',
  },
});

// 2. Upload file
await fetch(presigned.url, {
  method: 'PUT',
  headers: { 'Content-Type': 'image/jpeg' },
  body: imageFile, // File object
});
```
{% endtab %}
{% endtabs %}

{% hint style="warning" %}
⚠️ The Presigned URL is valid for **15 minutes** after issuance. If it expires, request a new one.
{% endhint %}

## Step 4: List Products

Browse registered products with various conditions.

{% tabs %}
{% tab title="MCP (AI Tools)" %}
{% hint style="success" %}
✅ **Try saying this to AI**

"Show me products in the Clothing category that are 50,000 won or less, sorted by lowest price first."
{% endhint %}

The AI queries products matching the conditions and shows the results.
{% endtab %}

{% tab title="Console + REST API" %}

### All Products

```bash
curl -X GET "https://api-client.bkend.ai/v1/data/products" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}"
```

### Category Filter

```bash
curl -X GET "https://api-client.bkend.ai/v1/data/products?andFilters=%7B%22category%22%3A%22Clothing%22%7D" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}"
```

### Sort (Price Ascending)

```bash
curl -X GET "https://api-client.bkend.ai/v1/data/products?sortBy=price&sortDirection=asc" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}"
```

**bkendFetch example:**

```javascript
// Product list by category
const products = await bkendFetch(
  `/v1/data/products?andFilters=${encodeURIComponent(JSON.stringify({ category: "Clothing" }))}&sortBy=price&sortDirection=asc`
);

console.log('Product list:', products.items);
console.log('Total count:', products.pagination.total);
```

## Step 8: Inventory Management

Deduct stock when orders come in, and add stock when restocking.

{% tabs %}
{% tab title="MCP (AI Tools)" %}
{% hint style="success" %}
✅ **Try saying this to AI**

"Show me the list of products with 10 or fewer items in stock."
{% endhint %}

The AI finds and shows products with low stock among active products.

{% hint style="success" %}
✅ **When restocking:**

"Increase the stock of Premium Cotton T-Shirt to 150."
{% endhint %}
{% endtab %}

{% tab title="Console + REST API" %}

### Update Stock

```bash
curl -X PATCH https://api-client.bkend.ai/v1/data/products/{product_id} \
  -H "Content-Type: application/json" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}" \
  -d '{
    "stock": 80
  }'
```

### Query Low-Stock Products

```bash
curl -X GET "https://api-client.bkend.ai/v1/data/products?andFilters=%7B%22isActive%22%3Atrue%7D" \
  -H "X-API-Key: {pk_publishable_key}" \
  -H "Authorization: Bearer {accessToken}"
```

**bkendFetch example:**

```javascript
// Query active products, then filter by stock on client side
const activeProducts = await bkendFetch(
  `/v1/data/products?andFilters=${encodeURIComponent(JSON.stringify({ isActive: true }))}`
);

const lowStock = activeProducts.items.filter(p => p.stock <= 10);
console.log('Low stock products:', lowStock);
```
{% endtab %}
{% endtabs %}

{% hint style="info" %}
💡 When stock reaches 0, it's recommended to set `isActive` to `false` to mark the product as sold out.
{% endhint %}

## Error Handling

| HTTP Status | Error Code | Description | Solution |
|:-----------:|------------|-------------|----------|
| 400 | `data/validation-error` | Required field missing | Check name, description, price, category, stock |
| 401 | `common/authentication-required` | Authentication failed | Check Access Token |
| 404 | `data/not-found` | Product not found | Check product ID |
| 413 | `file/too-large` | File size exceeded | Reduce image size |

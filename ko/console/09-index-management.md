# 인덱스 관리

{% hint style="info" %}
💡 인덱스를 생성하면 데이터 조회 성능을 크게 향상시킬 수 있습니다.
{% endhint %}

## 개요

인덱스는 테이블의 특정 필드에 대한 조회 속도를 최적화하는 데이터 구조입니다. 자주 검색하거나 정렬하는 필드에 인덱스를 생성하세요.

***

## 인덱스 목록 조회하기

1. **데이터베이스** 페이지에서 테이블을 클릭하세요.
2. **Indexes** 탭을 클릭하세요.

<!-- 📸 IMG: 인덱스 목록 화면 -->
![인덱스 목록](../.gitbook/assets/2026-02/console-index-list.png)

### 인덱스 목록 컬럼

| 컬럼 | 설명 |
|------|------|
| 이름 | 인덱스 이름 (예: `email_1`) |
| 필드 | 인덱스를 구성하는 필드와 정렬 순서 (ASC / DESC) |
| Unique | 고유 인덱스 여부 (✅ 체크 아이콘 / —) |
| Sparse | 희소 인덱스 여부 (✅ 체크 아이콘 / —) |
| 액션 | 삭제 (시스템 인덱스 제외) |

{% hint style="warning" %}
⚠️ `_id_` 인덱스는 시스템 기본 인덱스로, 삭제할 수 없습니다.
{% endhint %}

***

## 인덱스 생성하기

1. **Add Index** 버튼을 클릭하세요.
2. 인덱스를 구성할 필드를 선택하세요.
3. 각 필드의 정렬 순서를 지정하세요 (ASC / DESC).
4. **생성**을 클릭하세요.

<!-- 📸 IMG: 인덱스 생성 모달 -->
![인덱스 생성](../.gitbook/assets/2026-02/console-index-create.png)

### 인덱스 옵션

인덱스 생성 폼에서 **고급 옵션**을 펼치면 다음 옵션을 설정할 수 있습니다.

| 옵션 | 설명 |
|------|------|
| **Unique** | 활성화하면 해당 필드 조합에 중복 값을 허용하지 않습니다 |
| **Sparse** | 활성화하면 해당 필드가 존재하는 문서만 인덱싱합니다 |

{% hint style="info" %}
💡 여러 필드를 추가하면 복합(Compound) 인덱스가 됩니다. 여러 필드를 동시에 검색/정렬하는 쿼리의 성능을 향상시킬 수 있습니다.
{% endhint %}

***

## 인덱스 삭제하기

1. 삭제할 인덱스의 **삭제** 버튼을 클릭하세요.
2. 확인 대화상자에서 **삭제**를 클릭하세요.

{% hint style="warning" %}
⚠️ 시스템 인덱스와 시스템 필드로만 구성된 인덱스는 삭제할 수 없습니다.
{% endhint %}

***

## 인덱스 버전 히스토리

인덱스를 변경할 때마다 새 버전이 생성됩니다. **History** 버튼을 클릭하면 변경 이력을 확인하고 이전 버전으로 롤백할 수 있습니다.

{% hint style="info" %}
💡 AI 도구에서도 인덱스를 관리할 수 있습니다. 자세한 내용은 [Database MCP 도구](../ai-tools/12-mcp-db-tools.md)를 참고하세요.
{% endhint %}

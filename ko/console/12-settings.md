# 프로젝트 설정

{% hint style="info" %}
💡 프로젝트의 기본 정보와 설정을 관리하는 방법을 안내합니다.
{% endhint %}

## 개요

프로젝트 설정 페이지에서 Project ID 확인, 프로젝트 정보 수정, 프로젝트 삭제 등을 수행합니다.

***

## 설정 페이지 열기

프로젝트 레벨 사이드바에서 **프로젝트 설정**을 클릭하세요.

***

## 프로젝트 정보

설정 페이지 상단에 프로젝트의 주요 정보가 표시됩니다.

| 필드 | 설명 |
|------|------|
| **Project ID** | 자동 생성된 고유 식별자 (클릭하여 복사) |
| **프로젝트 Slug** | URL 식별자 |
| **클라우드 제공자** | 클라우드 제공자 (예: AWS) |
| **리전** | 배포 리전 (예: ap-northeast-2) |
| **생성일** | 프로젝트 생성 일시 |

{% hint style="info" %}
💡 Project ID는 자동 생성된 고유 식별자입니다. 변경할 수 없습니다. Publishable Key(`pk_` 접두사)에 프로젝트 ID와 환경 정보가 포함되어 있으므로, `X-API-Key` 헤더만으로 프로젝트를 식별합니다.
{% endhint %}

```bash
-H "X-API-Key: {pk_publishable_key}"
```

***

## 프로젝트 설정 수정하기

| 필드 | 수정 가능 | 설명 |
|------|:--------:|------|
| **프로젝트 이름** | ✅ | 표시 이름 변경 |
| **설명** | ✅ | 프로젝트 설명 변경 |
| **프로젝트 Slug** | ❌ | 생성 후 변경 불가 |
| **클라우드 제공자** | ❌ | 생성 후 변경 불가 |
| **리전** | ❌ | 생성 후 변경 불가 |
| **Project ID** | ❌ | 자동 생성, 변경 불가 |

***

## CORS (허용 출처)

프로젝트 레벨 사이드바에서 **CORS**를 클릭하면 교차 출처 API 요청에 대한 허용 출처를 관리할 수 있습니다.

1. 출처 URL을 입력하고 (예: `https://my-app.com`) **추가**를 클릭하세요.
2. 허용 목록에 출처가 추가됩니다.

| 규칙 | 설명 |
|------|------|
| **최대 수** | 프로젝트당 최대 3개 |
| **형식** | 프로토콜을 포함한 유효한 URL (예: `https://`) |
| **와일드카드** | 와일드카드(`*`)는 허용되지 않습니다 |
| **localhost** | 개발용으로 `http://localhost` 출처가 허용됩니다 |

출처를 삭제하려면 옆에 있는 **삭제** 버튼을 클릭하세요.

{% hint style="warning" %}
⚠️ 허용 목록에 없는 출처에서의 API 요청은 브라우저의 CORS 정책에 의해 차단됩니다.
{% endhint %}

***

## 프로젝트 삭제하기

{% hint style="danger" %}
🚨 **위험** — 프로젝트를 삭제하면 모든 환경, 테이블, User, 파일, API Key가 영구 삭제됩니다. 이 작업은 되돌릴 수 없습니다.
{% endhint %}

1. 설정 페이지 하단의 **프로젝트 삭제** 섹션으로 이동하세요.
2. **삭제** 버튼을 클릭하세요.
3. 프로젝트 이름을 입력하고 **삭제 확인**을 클릭하세요.
